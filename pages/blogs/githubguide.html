üîç What is GitHub?

GitHub is a web-based platform for version control and collaboration. It allows developers to store, manage, and track changes to their code using a system called Git, which was created by Linus Torvalds, the creator of Linux.

  • Share code
  • Review each other's work

üß© Key Concepts You Should Know

1. Git vs. GitHub

Git is the version control system that tracks code changes.
GitHub is the cloud-based hosting service that uses Git under the hood, with added features like a GUI, issue tracking, and collaboration tools.

2. Repository (Repo)

A repository is like a folder that stores your code, documentation, and history of changes. You can create:

  • Public repos (visible to everyone)
  • Private repos (visible only to selected users)

3. Commit

A snapshot of your changes. Every time you make edits and save them with a message, that‚Äôs a commit.

4. Branch

A branch lets you work on new features or bug fixes without affecting the main (production) code. The default branch is usually called main or master.

5. Merge & Pull Requests (PR)

When your feature is ready, you merge it back into the main branch. You do this through a pull request, which lets others review and discuss your code before it becomes part of the project.

6. Fork

A copy of someone else‚Äôs repository under your GitHub account. You can make changes and suggest improvements via pull requests.

7. Clone

A local copy of the repository on your computer. You can edit code offline and push changes back to GitHub.

8. Issues

Used for tracking bugs, feature requests, or questions. Each issue can be labeled, assigned, and commented on.
