GitHub is a web-based platform for version control and collaboration. It allows developers to store, manage, and track changes to their code using a system called Git, which was created by Linus Torvalds.

  • Share code
  • Review each other's work
  • Track issues and bugs

üß© Key Concepts You Should Know

1. Git vs. GitHub

Git is the version control system.
GitHub is a cloud-based host using Git with collaboration tools.

2. Repository (Repo)

A repository stores your code, docs, and version history.

  • Public repos - visible to all
  • Private repos - limited access

3. Commit

A snapshot of changes. Each save with a message is a commit.

4. Branch

Allows working on features without affecting the main code. Default branch is main or master.

5. Merge & Pull Requests (PR)

Merge branches via pull requests. Others can review your code before it merges into the main branch.

6. Fork

Copy someone‚Äôs repo to your account. Propose changes through pull requests.

7. Clone

Download the repo locally to work offline.

8. Issues

Used to track bugs, requests, or questions. They can be assigned, labeled, and commented on.
